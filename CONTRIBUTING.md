# Contributing to God's Own Trails 🌴

Thank you for your interest in contributing to God's Own Trails! We welcome contributions from the community. This document provides guidelines and instructions for contributing.

## Ways to Contribute

- **Add new destinations** - Share travel guides for new Kerala destinations
- **Improve content** - Enhance existing destination descriptions, tips, and information
- **Fix bugs** - Report and fix issues
- **Improve design** - Suggest and implement UI/UX improvements
- **Add features** - Implement new interactive elements
- **Documentation** - Improve README and guides

## Getting Started

### 1. Fork the Repository
Click the "Fork" button in the top-right corner of the GitHub repository.

### 2. Clone Your Fork
```bash
git clone https://github.com/YOUR-USERNAME/travel-blog.git
cd travel-blog
```

### 3. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names:
- `feature/add-destination-name` - For new destinations
- `fix/issue-description` - For bug fixes
- `docs/documentation-update` - For documentation
- `style/ui-improvements` - For design improvements

### 4. Make Your Changes
Make your changes to the relevant files. For new destinations:
1. Create a new HTML file: `destination-name.html`
2. Add a link in `index.html`
3. Include a destination image: `destination-name.jpg`
4. Follow the existing HTML structure and styling

### 5. Test Your Changes
- Open `index.html` in your browser using Live Server
- Check all links work correctly
- Ensure responsive design works on mobile, tablet, and desktop
- Verify no console errors

### 6. Commit Your Changes
```bash
git add .
git commit -m "Add: Brief description of changes"
```

Use clear commit messages:
- `Add: New destination page for Thekkadi`
- `Fix: Navigation link bug on mobile`
- `Improve: Image optimization`
- `Docs: Update README with new features`

### 7. Push to Your Fork
```bash
git push origin feature/your-feature-name
```

### 8. Create a Pull Request
1. Go to your fork on GitHub
2. Click "Compare & pull request"
3. Add a clear title and description
4. Reference any related issues
5. Submit the pull request

## Code Guidelines

### HTML
- Use semantic HTML5 elements
- Keep code properly indented
- Add descriptive comments for complex sections
- Follow the existing file structure

### CSS
- Use consistent class naming conventions
- Keep styles organized by component
- Avoid inline styles
- Use CSS variables for colors and common values
- Ensure mobile responsiveness

### Content
- Write clear, descriptive text
- Include practical travel information
- Add proper attribution for images
- Keep content accurate and up-to-date
- Use engaging language

## Destination Page Template

When adding a new destination, include these sections:
1. **Header** - Destination name and overview
2. **Quick Facts** - Location, best time to visit, distance from major cities
3. **Attractions** - Main attractions and activities
4. **Getting There** - Transportation options
5. **Best Time to Visit** - Seasonal information
6. **Accommodation** - Lodging options
7. **Local Cuisine** - Traditional food recommendations
8. **Travel Tips** - Practical advice
9. **Gallery** - Destination images

## Pull Request Process

1. **Ensure quality**
   - All links are functional
   - No console errors
   - Cross-browser compatible
   - Mobile responsive

2. **Update documentation**
   - Update README if adding features
   - Document breaking changes
   - Add comments to complex code

3. **Respond to feedback**
   - Be open to suggestions
   - Make requested changes
   - Keep communication respectful

4. **Merge**
   - Project maintainers will review
   - Once approved, your PR will be merged
   - Your contribution will be credited

## Reporting Issues

Found a bug or have a suggestion? Please create an issue:

1. Go to the Issues tab
2. Click "New Issue"
3. Use a clear, descriptive title
4. Provide:
   - Description of the issue
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots if applicable
   - Your browser/device info

## Community Standards

- Be respectful and inclusive
- Provide constructive feedback
- Help other contributors
- Follow the project's code of conduct
- Report inappropriate behavior to maintainers

## Questions?

- Check existing issues and discussions
- Review the README
- Comment on relevant issues
- Ask questions in pull requests

## Recognition

Contributors will be:
- Listed in the README
- Credited in commit messages
- Acknowledged in release notes

Thank you for contributing to God's Own Trails! 🌴✈️
